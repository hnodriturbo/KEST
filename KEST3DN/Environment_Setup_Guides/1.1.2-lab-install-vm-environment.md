# 1.1.2 — Install the Virtual Machine Lab Environment (Condensed, Do-This Guide)

> **What you have**: `DEVASC_VM.ova` (a prebuilt VirtualBox appliance).
> **What *not* to do**: Don’t use “New” → ISO. OVA ≠ ISO. Use **Import Appliance** instead.

---

## Part 1 — Prepare your computer & import the VM
- ✅ **Check resources**: You should have ≥ **4 GB RAM** free and **15 GB** disk available.
- ✅ **Install VirtualBox**:
  1. Go to `virtualbox.org` → **Downloads** → install for your OS.
  2. Accept defaults; finish setup; launch VirtualBox.
- ✅ **Import the DEVASC VM (.ova)**:
  1. In VirtualBox, **File → Import Appliance…**
  2. **Source**: *Local File System* → browse to your `DEVASC_VM.ova` → **Open**.
  3. Review settings (you can adjust RAM/CPUs later if needed) → **Import** and wait.

> **If you only see an ISO chooser**: You probably clicked **New**. Cancel it and use **File → Import Appliance…** instead.

---

## Part 2 — First boot & GUI tour
1. Select the imported **DEVASC VM** → **Start** (first boot takes a bit).
2. **Packet Tracer EULA** (first boot only):
   - Arrow keys to scroll → **Right Arrow** to select `<OK>`.
   - **Space Bar** to continue to the Agreement screen.
   - **Left Arrow** to select `<I Agree>`.
3. When the Ubuntu desktop loads, briefly open these so you know where they are:
   - **Terminal**, **VS Code**, **Packet Tracer**, **Chromium**, **Postman**.
4. Explore **Menu → Places / System / All applications** (this VM is isolated—feel free to click around).

---

## Part 3 — Create course accounts (quick setup)
- **DevNet**: `developer.cisco.com` → **SIGN UP FREE** → pick a login method you prefer; finish signup.
- **GitHub**: `github.com` → **Sign up** → username, email, password → **Join a free plan** → **Complete setup**.
- **Webex**: `webex.com` → **Start for Free / Sign up now** → enter email → follow prompts.

*(You can also use Webex in a browser if you don’t want the desktop app.)*

---

## Part 4 — Shut down or save state properly
- **Save state (recommended while labbing)**:
  - VM window menu **VirtualBox → File → Close…** → **Save the machine state** → **OK**.
  - Next start resumes exactly where you left off.
- **Other close options**:
  - **Send the shutdown signal** (like pressing a PC power button).
  - **Power off the machine** (like pulling the plug—avoid unless frozen).

- **CLI alternatives inside the VM** (use Terminal):

```bash
# Graceful shutdown now (halts the VM)
sudo shutdown -h now

# Reboot the VM
sudo reboot
```

---

## Troubleshooting (fast fixes)
- **Only “ISO” option appears**: Use **File → Import Appliance…** (don’t use “New”).
- **Import error**: Ensure the `.ova` finished downloading (re-download if corrupted).
- **No network in VM**: In VM **Settings → Network**, ensure **Adapter 1: NAT** is enabled.
- **Slow VM**: After import, try **Settings → System**: increase **Base Memory** and **Processor** (if your host allows it).

---

## You’re done when…
- VirtualBox shows a running **DEVASC VM** you can log into.
- You can open Terminal/VS Code/Packet Tracer/Chromium/Postman.
- Your **DevNet**, **GitHub**, and **Webex** accounts exist and you can sign in.
